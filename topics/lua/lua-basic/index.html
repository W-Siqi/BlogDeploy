    前置知识: 了解过一门静态语言（C/C++/C#/..)和一门动态语言(javascipt/python/..)

    lua介绍

    lua的设计初衷就是作为拓展式语言(extension programming language). 也就是说一般会有一个宿主语言，然后这个宿主语言在一些地方执行一段lua的代码。
    Unity C# 游戏开发就是一个例子。C#是宿主语言，而一些需要热更新,灵活变动的功能模块就用lua写，运行的时候C#调用lua。

    变量与值

    如何看待和定义变量和值是作为一门语言很核心的一概念。
    Lua里面变量是没有类型的，但是值有类型。一个例子就是，a这变量一开始值的类型为string，但运行这运行这a的值的类型可能就变成boolean了。

    八大类型

    • nil: 类似于其他语言的none,null,nullptr
    • boolean: 布尔值
    • string: 字符串
    • number: lua里面只有双精度浮点数(对，没有int！)
    • table: lua中唯一的数据结构。
    • thread: 线程
    • userdata: 它是指宿主语言内存里的一段裸数据，只有宿主语言通过C-API来操作访问userdata，lua语言本身是没有权限操作的(可以看出lua作为一本拓展性语言，在语言层面对嵌入宿主语言进行了支持)
    • function： lua里面函数式“一等公民”,他可以像number这种值一样当做变量赋值，传参..

    值和引用

    lua里table是传引用的，string和number都是传值的。
    function在lua里面也是”传引用“的，也就是 a = "function" a变量本身存的是指向b的引用

    Table

    很多语言都内置array,dictionary,list这些数据结构。在lua里面，只有一种数据结构:table. 因为这个table就可以拿当arrays,dict, queues等等数据结构来用。

    所以table本质上是个什么东西？为什么能拿来用这么多？
    我们可以把它看做一个key可以是number或string的dictionary。你把它的key都是1，2，3，4那就是数组了。

    实际上，你可以把array看做索引规则更严格的dictionary，比如array的索引必须是数字，可能必须遵循0,1,2,3这样顺序的添加。Lua选择用table统一array和dictionary，代价就是很难约束table的访问规则成纯数组，比如你可以1，2，4，中间的3下标是没东西的。而且不能像其他语言，比如C++/C#，一样一下子分配一整段内存，只能动态地去分配内存，效率少可能会吃亏。
    但我能理解lua这么做的原因，因为他就想做一个小而精的嵌入型语言，灵活，统一，简洁在table这里就很能体现他的理念。

    Metatable

    首先这个metatable是一个lua里面的table，其次我们可以给任意table绑定一个metatable(默认是不绑定)，最后就是这个metatable定义了这些特殊的键值，这些特殊的键值可以改变这个table系统层面上的行为

    lua的全局变量

    lua的全局变量也是用一个table来实现的，这个table就是特殊的__G。因为他是table，不仅仅一位置我们可以通过键值对全局变量进行操作，更意味着我们可以通过设置__G的metatable来定义一些全局变量的行为。

    lua的面向对象

    lua是通过table来模拟面向对象的object的。他的object实质是一个成员可灵活变的一个table。
    object的成员函数，同function来定义，只要定义的时候加个table的名字作为前缀就ok

    垃圾回收

    lua使用的算法是 mark-and-sweep(追踪垃圾回收)
    一个参考： https://www.geeksforgeeks.org/mark-and-sweep-garbage-collection-algorithm/

    __gc

    因为lua作为嵌入语言，可会拥有一些宿主语言的资源，或者网络端口，数据库链接这种lua的垃圾回收无法处理的资源。这个时候你就可以在__gc这个metafunction里写自己的回收逻辑。 __gc会在被回收前被调用。

    Weak table

    任何垃圾回收都无法做到“真正懂你”，程序员很多时候是需要额外干预的。 比如有的垃圾回收器，在循环引用的时候会回收不掉。lua虽然gc虽然没有循环引用的困扰，但是在其他情形下也是需要辅助的。这个参与辅助的机制局势weaktable。
    (详细见: http://www.lua.org/pil/17.html )

    其他

    C-API 与虚拟栈

    lua作为嵌入型语言，宿主语言调用lua，实际上就调用lua提供的C-API。
    这种程序交互要解决的一个头号问题就是：我的数据（参数/返回值）怎么传输？
    lua是用一种栈的结构去模拟这种交互，从C-API的风格其实就很明显，基本都是对栈进行操作的API，比如push,pop... 采用虚拟栈这个选择本身也是好理解的，因为大多计算机语言本身的运行实现就是基于自己的一个调用堆栈。

    错误处理

    lua作为嵌入型语言，都是宿主语言用C 调用lua，一旦lua代码出错，就立即返回C。
    如果要捕获error，宿主语言需要用p_call这个C-API

    Coroutine

    lua 也支持coroutine协程。类似unity里的协程。
    给coroutine一个定义就是，一个coroutine代表的是一次时间上不连续的线性执行。他和函数一样是一句句地执行下去，但是不是一口气执行完才返回，程序员可以显示地用yield挂起，并且制定好挂起结束的时机，好了之后再返回来继续执行。

    coroutine 和 thread的区别，就是thread一般是同时执行的，而coroutine同一时间只执行一个。
